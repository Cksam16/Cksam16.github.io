About me

Team 8 is a five-engineer collective from the Singapore Institute of Technology that treats integration as a first-class discipline. Working in ROS 1 (Melodic), we stitch together LIDAR, IMU and RGB-D feeds via tf and sensor_msgs, layer tight motion loops with ros_control, and unify autonomy through the classic navigation stack (move_base + AMCL + gmapping).
Our build system (catkin), runtime tooling (roslaunch, rosbag, RViz, Gazebo) and GitHub Actions CI pipeline form a continuous integration chain—from sensor drivers to cloud dashboards—so every commit is validated on real Wheeled Mobile Robots before it merges. The result: field-ready platforms that teams can operate locally or through secure remote tele-op without wrestling with mismatched nodes or flaky interfaces.
In short, we don’t just develop robotic components—we integrate them into coherent systems that behave reliably in the wild, straight out of the crate.

Projects

Autonomous Navigation and Obstacle Construction

Built an Obstacle course and programmed a ROS based Autonomous LIMO Robot to operate in a simulated indoor environment representing the Changi Airport.
• Designed and constructed the Obstacle course with high fidelity and clean workmanship, referencing real layouts for spatial accuracy of Changi Airport Terminal 3.
• Implemented SLAM using GMapping and Cartographer to dynamically build a map of the environment in real time.
• Used Dijkstra’s algorithm for global pathfinding, ensuring the robot selected the optimal route between arena entrances and key waypoints.
• Integrated AMCL for localisation, configured costmaps for obstacle avoidance, and used move_base for continuous path execution.
• Implemented Dynamic Obstacle Handling and Recovery Behaviours; Tested extensively in real and simulated conditions.
• Tuned LiDAR-based detection, built TF trees, and executed all configuration and deployment through Ubuntu CLI.
• Ensured seamless robot traversal between plots and aligned arena design with aesthetic and functional goals.

Litekit Robot

Developed a mobile robot for semi-autonomous navigation and object tracking using sensors and communication protocols.
• Controlled via XBee (Zigbee) wireless modules enabling reliable remote communication for telemetry and control.
• Integrated ultrasonic and ToF (Time-of-Flight) sensors for multi-range obstacle detection.
• Powered by the P1 microcontroller, managing real-time control logic and sensor data processing.
• Employed PixyCam for color-based object recognition and visual tracking tasks.
• Integrated mecanum wheels to enable omnidirectional movement for navigating tight spaces and dynamic path adjustments.
• Designed and wired an emergency stop mechanism for user safety and controlled shutdown in critical conditions.

TF2 TurtleBot Simulation

Built a ROS tf2 setup in turtlesim to simulate real-time leader-follower behavior using transform broadcasting.
• Used tf2 broadcaster to publish turtle1’s pose relative to the world frame.
• Implemented listener node to control turtle2 based on real-time transform data, enabling autonomous following.
• Verified system behavior using key ROS tools:
– tf_echo to monitor live distance and orientation between turtles
– view_frames to generate a static transform tree diagram
– RViz to visually inspect spatial frame positions and orientations
